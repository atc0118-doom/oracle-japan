# ORACLE JAPAN — 国内生活リスク指数(実験ツール)

ORACLE(世界リスクインテリジェンス)の日本国内特化版。JAPAN NOW(中立ニュース集約)とは
役割が異なり、こちらは「日本での暮らしに直結するリスクをスコア化する」のが目的。

## 設計判断

- **配管はORACLE本体から流用**: 10分キャッシュ、Upstash Redisによる24Hデルタ永続化(未設定でも
  無言でno-opする)、`isBaseline`による「ライブでないデータをライブと偽らない」フォールバック設計。
- **災害カテゴリだけ構造化データ駆動**: JMA公式警報(都道府県別・特別警報/警報/注意報の重み付け)
  と地震Atomフィード(震度5弱以上の非定時速報のみ)から算出。ここはキーワード推測ではなく公式デー
  タそのものなので、他4カテゴリより信頼度が高い。
- **他4カテゴリ(安全保障・感染症・インフラ・治安)はキーワード頻度**: ORACLE本体と同じ「記事件数
  を飽和カーブでスコア化」する手法。ニュース報道量の代理指標であって、検証済みの実測値ではない
  ことをUI上の免責文で明示している。
- **重み**: `Disaster 0.35 / Security 0.25 / Health 0.15 / Infrastructure 0.15 / PublicSafety 0.10`。
  「暮らしへの直接的な脅威」という趣旨上、災害の重みを意図的に最大にしている。ORACLE本体(世界の
  地政学的緊張)とは重みの思想自体が異なる。
- **JMA/NHK取得ロジックはJAPAN NOWから移植**: 47都道府県分のoffice走査、特別警報コード表、Hokkaido/
  Okinawaの表示グルーピング、定時降灰予報の除外ロジックなど。JAPAN NOW側で既に本番検証済みのロジ
  ックをそのまま再利用している。

## 未検証・今後の課題

- このサンドボックスはネットワークアクセスがないため、JMA/NHK/Google Newsへの実際の疎通確認は
  **未実施**。デプロイ後、まず`dataStatus`が`LIVE`になるか、`sourceError`に何も出ていないかを確認
  すること。
- キーワードリスト(`CATEGORY_KEYWORDS`)は初期案。実際の報道でどの程度誤検知/検知漏れがあるか、
  本番投入後にヘッドラインを見ながら調整が必要。
- Redis永続化(24Hデルタ)はORACLE本体と同じ環境変数(`KV_REST_API_URL`/`KV_REST_API_TOKEN`など)を
  流用する設計。ORACLE本体と同じUpstash DBを共用する場合はキー名(`oraclejapan:score_log`)が衝突
  しないことを確認済み。

## デプロイ

Vercelにこのフォルダをそのままpushすればよい(ORACLE本体・JAPAN NOWと同じ構成)。
Redis永続化を使う場合はVercel Marketplace経由でUpstash Redis統合を追加し、環境変数を注入する。

## テスト

```
npm test
```

ネットワーク呼び出しを伴わない純粋関数(スコア計算・重複排除・警報コード解釈など)のみをテスト
している。実際のfetch/パース部分はデプロイ後に本番データで確認する必要がある。
